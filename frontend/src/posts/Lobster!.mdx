I want to make a game.  
A tabletop style video game.  
I've actually been working on this very intermittently for a long time - never programming, only designing.
Now that I feel like I'm ready to take a stab at programming it, I'm confronted with the question - whats 
important to me? 

I think the answers to that are extensibility, performance, and stability. 

- I want the program to be extendable by users - mods, user generated content, etc. 
- It should perform better than roll20 and foundry vtt, enabling 120hz 4k on an igpu ideally. 
- I don't want to deal with the constant vulnerabilities and supply chain attacks of node. 

So here's some options 

1. Deno with phaser 
    - It seems like there's some really cool games made with this, and it has integrations with 
vite, svelte, vue, and other good ui frameworks so thats promising. 
    - The performance is poop, im getting 80fps using 70% of my 4070ti for a game that looks like this. 
    <ArticleImage src={delve_online}/>
    
2. [Lobster](https://aardappel.github.io/lobster/README_FIRST.html) - a programming language made specifically for games!
    - I tried to find games to demo to test this out but ...  I couldn't. Next. 
- Rust with Bevy 
    - Tons of [examples](https://bevy.org/examples-webgpu/transforms/align/)
    - Networking with [matchbox](https://johanhelsing.studio/posts/extreme-bevy) using webrtc. 
- Unity or Unreal? - lol nah
- Dioxus?
    - So it seems like dioxus can be embedded in bevy? But bevy has a native UI solution so I don't 
    think I'll bother unless it really sucks. 
    - Embedding bevy in dioxus would require using a webview and compiling bevy to wasm which seems like 
    itd kill the performance advantage over phaser.
